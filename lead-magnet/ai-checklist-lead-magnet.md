## TEIL 1: INFRASTRUKTUR & TOOLS (8 Punkte)

### Kommunikation
- [ ] Wir nutzen ein Team-Chat-Tool (Slack / Teams / Mattermost)
- [ ] Unser Chat-Tool hat eine API oder Webhook-Funktion
- [ ] Benachrichtigungen kommen zentral an einer Stelle an

### Automatisierung
- [ ] Wir kennen n8n, Zapier oder Make.com
- [ ] Mindestens ein Prozess läuft bereits automatisiert
- [ ] Wir haben Zugang zu einem API-fähigen KI-Dienst (OpenAI, Claude, etc.)

### Monitoring
- [ ] Wir wissen, wenn ein Server oder Service ausfällt (nicht erst durch Kunden)
- [ ] Wir haben ein Dashboard für den Systemstatus

## TEIL 2: PROZESSE & WORKFLOWS (8 Punkte)

### Wiederkehrende Aufgaben
- [ ] Wir haben mindestens 3 Aufgaben, die wöchentlich gleich ablaufen

- [ ] Meeting-Protokolle werden manuell geschrieben
- [ ] Code-Reviews folgen einem festen Schema
- [ ] Neue Aufgaben werden manuell zugewiesen und nachverfolgt

### Daten & Berichte
- [ ] Wir erstellen regelmäßige Reports aus verschiedenen Datenquellen
- [ ] Alerts und Benachrichtigungen werden manuell weitergeleitet

## BONUS: DIE 5 HÄUFIGSTEN FEHLER BEI KI-PROJEKTEN

1. **Zu groß starten** — Pilot mit einem Use Case, nicht mit fünf gleichzeitig
2. **Kein Fallback** — Jeder Agent braucht einen manuellen Fallback-Plan
3. **Team nicht einbinden** — Agents, die niemand nutzt, scheitern immer
4. **Kein Monitoring** — Wenn der Agent fehlerhaft arbeitet, muss das auffallen
5. **DSGVO ignorieren** — Klären Sie vorher: Welche Daten darf der Agent verarbeiten?
